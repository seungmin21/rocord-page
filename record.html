Error: ENOENT: no such file or directory, stat 'C:\Users\Administrator\Desktop\Web-IA\index.html'

이 에러는 express 방식으로 GET 요청 방식을 작성 하던 도중 발생했던 에러입니다. 이 에러를 해석해보면 경로에 파일이 없다는 내용입니다. 이 에러의 원인 분석은 path.join()과 관련이 깊습니다. "path"는 node.js의 내장모듈이며 path는 사용자가 파일/폴더/디렉토리의 경로를 간편하게 해주는 모듈입니다. path.join()은 인자를 제대로 작성하지 않으면 경로를 잡아주지 못합니다. 그렇기 때문에 경로 관련 작성이 포인트입니다.

Web Application IA의 코어 파일 구성을 보면 서버: routes.js : HTTP 라우팅을 처리하는 파일 에서 라우팅에 대해 알아보았습니다. "라우팅"이란 적절한 네트워크로 패킷을 전송하는 것인데 좀 더 말해보면 네트워크에서 최적의 경로를 선택하는 것입니다. 여기서 "패킷"은 데이터를 효율적으로 전송하는 통신입니다. 비유를 들어보면 30명이 되는 반에서 일일히 가정방문을 하는 대신 가정통신문으로 대체하는 것입니다.
